# Video-Generator 上线验收 Checklist

> **文档版本**: v1.0  
> **适用版本**: video-generator v3.0+  
> **验收目标**: 确保用户体验优秀、功能稳定、错误提示友好

---

## 📋 上线前人工验收清单

本清单聚焦于**难以通过自动化脚本覆盖的用户体验类问题**，必须在上线前由人工逐项验收。

---

### ✅ Checklist 1: 错误提示用户友好度 (P0 - 必须通过)

**验收目标**: 确保所有错误提示对小白用户友好，不使用技术术语

**验收步骤**:

1. **故意触发各种错误**，检查错误提示：
   - [ ] 空Prompt → 提示："请输入图片描述哦～"
   - [ ] API超时 → 提示："网络有点慢，请稍后再试～"
   - [ ] API限流 → 提示："使用人数过多，请稍等片刻～"
   - [ ] 余额不足 → 提示："API余额不足，请联系管理员充值"
   - [ ] 文件不存在 → 提示："找不到图片文件，请检查路径是否正确"

2. **验证错误提示不含技术术语**：
   - [ ] 不包含 "Traceback", "Exception", "Error 500" 等
   - [ ] 不包含代码堆栈信息
   - [ ] 使用 emoji 增强可读性（如 ❌✅⚠️）

3. **验证错误提示提供解决建议**：
   - [ ] 每个错误都附带"你可以尝试：" + 具体建议
   - [ ] 建议步骤清晰、可操作

**通过标准**: 所有错误提示都像"人话"，小白用户能看懂并知道如何解决

---

### ✅ Checklist 2: 生成进度反馈清晰度 (P0 - 必须通过)

**验收目标**: 确保生成过程中的进度反馈清晰、准确、不卡顿

**验收步骤**:

1. **测试图片生成进度反馈**：
   - [ ] 开始生成时显示："正在用AI画图中..." 
   - [ ] 显示当前使用的模型名称（如 "使用FLUX模型生成中..."）
   - [ ] 显示预估剩余时间（如 "预计需要15秒..."）
   - [ ] 完成后显示："✅ 图片生成完成！"

2. **测试视频生成进度反馈**（异步轮询）：
   - [ ] 提交任务后显示："任务已提交，正在排队..."
   - [ ] 轮询时显示："任务处理中... (已等待 10秒)"
   - [ ] 显示进度百分比（如 "进度：45%"）
   - [ ] 完成后显示："✅ 视频生成完成！"

3. **验证进度反馈不卡顿**：
   - [ ] 进度更新流畅（每秒更新1次）
   - [ ] 不阻塞用户其他操作（如果是UI应用）
   - [ ] 进度消息清晰（不使用 "Processing..." 这种模糊表述）

**通过标准**: 用户能准确知道当前进度，不会因为"不知道要等多久"而焦虑

---

### ✅ Checklist 3: UI交互流畅度 (P1 - 强烈建议通过)

**验收目标**: 确保UI交互不卡顿、响应及时

**验收步骤**:

1. **测试按钮响应速度**：
   - [ ] 点击"生成"按钮后立即禁用（防止重复点击）
   - [ ] 显示loading动画（旋转图标或进度条）
   - [ ] 按钮禁用状态持续到生成完成

2. **测试页面切换流畅度**：
   - [ ] 小白模式 ↔ 专业模式切换无延迟
   - [ ] 不出现白屏或闪烁
   - [ ] 切换后保留之前的输入内容

3. **测试文件上传体验**：
   - [ ] 拖拽上传响应及时
   - [ ] 显示上传进度（百分比或进度条）
   - [ ] 上传完成后显示预览图

4. **验证长时间操作的反馈**：
   - [ ] 视频生成（可能需要几分钟）时有明确提示："视频生成需要几分钟，你可以先做其他事情～"
   - [ ] 提供"后台运行"选项（如果是桌面应用）

**通过标准**: 所有交互都流畅，用户不会感到"卡"或"不响应"

---

### ✅ Checklist 4: 生成结果质量 (P0 - 必须通过)

**验收目标**: 确保生成的图片/视频质量符合用户预期

**验收步骤**:

1. **测试图片生成质量**：
   - [ ] 生成的图片清晰度足够（不低于512x512）
   - [ ] 图片内容与Prompt相关（不生成无关内容）
   - [ ] 不生成畸形图片（如多只眼睛、肢体扭曲）

2. **测试视频生成质量**：
   - [ ] 视频画面流畅（不低于24fps）
   - [ ] 音频与画面同步（不出现音画不同步）
   - [ ] 视频长度与旁白长度匹配

3. **测试数据可视化质量**：
   - [ ] 图表数据准确无误
   - [ ] 动画过渡流畅
   - [ ] 文字标签清晰可读

**通过标准**: 生成结果质量达到"可用"标准，不需要大量人工修正

---

### ✅ Checklist 5: 小白模式易用性 (P0 - 必须通过)

**验收目标**: 确保小白模式真正"小白友好"

**验收步骤**:

1. **测试默认参数合理性**：
   - [ ] 小白模式使用高质量默认模型（flux 或 sdxl）
   - [ ] 默认分辨率适合分享（1920x1080 或 1024x1024）
   - [ ] 不需要用户手动调整任何参数就能生成高质量结果

2. **测试引导文案清晰度**：
   - [ ] 输入框有占位符提示："描述你想生成的图片，如：一只可爱的猫"
   - [ ] 有"_example_"或"试试这个"按钮，提供示例Prompt
   - [ ] 生成完成后有"继续生成"按钮，鼓励用户尝试

3. **验证错误处理友好**：
   - [ ] 小白模式下不显示技术错误详情
   - [ ] 只显示用户友好的错误提示 + 解决建议

**通过标准**: 一个完全不懂AI的小白用户，能在3分钟内成功生成第一张图片

---

### ✅ Checklist 6: 专业模式灵活性 (P1 - 强烈建议通过)

**验收目标**: 确保专业模式提供足够的自定义选项

**验收步骤**:

1. **测试模型选择灵活性**：
   - [ ] 能切换所有支持的模型（flux, sdxl, sd15, dalle2, dalle3）
   - [ ] 显示每个模型的特点（速度/质量/费用）
   - [ ] 能保存常用模型为默认

2. **测试参数调整灵活性**：
   - [ ] 能调整分辨率、引导系数、推理步数
   - [ ] 参数调整有实时预览（如分辨率调整后立即显示图片尺寸）
   - [ ] 能保存参数为预设（如"高质量预设"、"快速预览预设"）

3. **测试高级功能可用性**：
   - [ ] 能使用负面提示词（negative prompt）
   - [ ] 能调整种子值（seed）以复现结果
   - [ ] 能批量生成（一次生成多张图片）

**通过标准**: 专业用户能精细化控制生成过程，实现想要的效果

---

### ✅ Checklist 7: 文档和教程完整性 (P1 - 强烈建议通过)

**验收目标**: 确保用户能快速上手

**验收步骤**:

1. **测试快速开始教程**：
   - [ ] 有"5分钟快速上手"教程
   - [ ] 教程包含动图或视频演示
   - [ ] 教程覆盖最核心的3个功能：生成图片、生成视频、AI图片生成

2. **测试API文档完整性**（如果是开发者工具）：
   - [ ] 有完整的API参考文档
   - [ ] 每个API都有示例代码
   - [ ] 有Postman/Insomnia 集合（方便测试）

3. **测试错误处理文档**：
   - [ ] 有"常见错误及解决方法"文档
   - [ ] 每个错误都有错误码 + 详细解决步骤
   - [ ] 提供"一键诊断"工具（自动检测环境问题）

**通过标准**: 用户90%的问题能通过文档自行解决，不需要联系客服

---

### ✅ Checklist 8: 性能和稳定性 (P0 - 必须通过)

**验收目标**: 确保系统在高负载下依然稳定

**验收步骤**:

1. **测试并发处理能力**：
   - [ ] 同时提交10个生成请求，所有请求都能成功
   - [ ] 系统资源（CPU/内存/磁盘）使用合理，不出现资源耗尽

2. **测试长时间运行稳定性**：
   - [ ] 连续运行24小时，生成100个任务，无内存泄漏
   - [ ] 临时文件自动清理，磁盘空间不持续增长

3. **测试异常恢复能力**：
   - [ ] 生成过程中断网，恢复后能继续（或优雅失败）
   - [ ] 生成过程中程序崩溃，重启后能恢复未完成的任务

**通过标准**: 系统能稳定处理真实生产环境的工作负载
